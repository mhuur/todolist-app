🔗 Partage avec d'autres comptes Google

Permet à plusieurs comptes Google d'accéder aux mêmes recettes (utile pour un couple, une famille, etc.). Chaque personne se connecte avec son propre compte mais voit les recettes communes.

💡 Les deux comptes doivent utiliser exactement le même nom. Choisis-le d'abord, puis active le mode ci-dessous.
Quand c'est activé, l'app lit/écrit dans l'espace commun ci-dessus au lieu de ton espace personnel.
    💡 Ton UID y est ajouté automatiquement. Pour obtenir l'UID de l'autre personne : qu'elle se connecte une fois à l'app depuis son propre compte Google, son UID apparaîtra dans Firebase Console → Authentication → Users.

    ⚙ Configurer Firebase
    📖 Étapes pour créer ta config Firebase (cliquer pour voir)
    1. Va sur console.firebase.google.com et connecte-toi avec ton compte Google
    2. Clique Ajouter un projet (ou "Add project") → donne-lui un nom (ex: mes-recettes) → Désactive Google Analytics → Créer
    3. Une fois dans le projet : clique sur l'icône </> (Web) au milieu de la page pour ajouter une application web. Donne-lui un surnom (ex: recettes-web) → Enregistrer
    4. Firebase affiche un bloc de code firebaseConfig = {...} — copie tout l'objet (de { à }) et colle-le dans la zone ci-dessous
    5. Dans le menu à gauche, va dans Build → Authentication → Get started → onglet Sign-in method → Google → activer → Enregistrer
    6. Dans Authentication → Settings → Authorized domains, vérifie que mhuur.github.io est listé (sinon ajoute-le)
    7. Dans le menu à gauche, va dans Build → Firestore Database → Create database → choisis un emplacement (europe-west3 p. ex.) → Start in production mode → Créer
    8. Dans l'onglet Rules de Firestore, remplace tout par les règles ci-dessous, puis Publier : 
      rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /users/{userId}/{document=**} {
      allow read, write: if request.auth != null 
                         && request.auth.uid == userId;
    }
  }
}
    💡 Tu peux coller soit l'objet entier { apiKey: ..., ... }, soit du JSON.
